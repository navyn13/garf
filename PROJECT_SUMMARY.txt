GARFIELD COIN - VIRAL GAME WEBSITE
===================================

PROJECT OVERVIEW
----------------
A complete meme coin website featuring an endless runner game where players collect dollars,
compete on daily leaderboards, and top 50 players win merchandise. Built with modern web3
integration using Solana wallet connectivity.

ARCHITECTURE
------------
Frontend:
- Next.js 16 with App Router
- TypeScript for type safety
- Tailwind CSS for styling
- Client-side game rendering
- Dynamic imports for SSR compatibility

Game Engine:
- Phaser 3 (canvas-based)
- Physics-based collision detection
- Procedural obstacle/dollar generation
- Progressive difficulty scaling

Backend:
- Next.js API routes
- MongoDB for data persistence
- RESTful API design
- Serverless-ready

Database Schema:
- scores: { wallet_address, score, duration, timestamp, date, created_at }
- rewards: { wallet_address, name, address, phone, rank, date, claimed_at }

API ENDPOINTS
-------------
POST /api/submit-score
  Request: { wallet_address, score, duration, timestamp }
  Response: { success, score, rank, isTopFifty }
  Validation: Anti-cheat checks, duplicate detection

GET /api/leaderboard
  Response: { leaderboard: [{ wallet_address, score, date, rank }] }
  Logic: Daily top 100, sorted by score

GET /api/stats
  Response: { playersToday, topScore }
  Updates: Real-time aggregation

POST /api/claim-reward
  Request: { wallet_address, name, address, phone, rank }
  Response: { success }
  Validation: Top 50 only, one claim per day

GAME MECHANICS
--------------
Controls:
- SPACE key or TAP screen to jump
- Automatic running (endless runner)

Scoring:
- Green circles (dollars) = +1 point
- Speed increases every 10 points
- Red obstacles = game over

Physics:
- Gravity: 1000
- Jump velocity: -500
- Scroll speed: 300 (base) + progressive increase

Spawn Rates:
- Obstacles: Every 2 seconds
- Dollars: Every 1.5 seconds
- Random Y positioning for dollars

ANTI-CHEAT DETAILS
------------------
Validation Rules:
1. Score >= 0 (no negative scores)
2. Score <= 10,000 (absolute maximum)
3. Duration >= 5 seconds (minimum game time)
4. Score/Duration <= 3 points/second (rate limit)
5. Timestamp within 60 seconds of submission
6. No duplicate submissions within 5 seconds

Server-Side:
- All validation happens on backend
- Client cannot bypass checks
- Invalid scores rejected with reason

DAILY RESET LOGIC
-----------------
Implementation:
- No cron job needed
- Date stored as midnight UTC timestamp
- Queries filter by date field
- Automatic reset when day changes

Example:
- Today: 2026-03-30 00:00:00
- Query: { date: new Date("2026-03-30T00:00:00Z") }
- Tomorrow: New date, new leaderboard

CUSTOMIZATION GUIDE

Game Difficulty:

- MAX_SCORE_PER_SECOND: default 3
- MIN_GAME_DURATION: default 5
- MAX_SCORE_ABSOLUTE: default 10000

Countdown Target:
- Edit components/Countdown.tsx
- Change targetDate (line 15)

Colors:
- Edit tailwind.config.ts

TROUBLESHOOTING
---------------
Issue: "window is not defined"
Fix: Game uses dynamic import with ssr: false

Issue: MongoDB connection error
Fix: Check MONGODB_URI format and network access

Issue: Wallet not connecting
Fix: Ensure running on https:// or localhost

Issue: Scores not submitting
Fix: Check browser console for validation errors
